Grid Component

Intro

The AEM Responsive Grid is entirely percentage based, typically with 12 columns. This means that a component or container set to 12 will be 100% of the width. A container set to 11 will be 91.6667%, a grid set to 6 will be 50% and so on. The AEM Responsive Grid supports 3 breakpoints: Default, Tablet, and Mobile. The breakpoints are configurable via the grid.less file when generating the grid over CSS. Containers in the grid do not stack automatically once different break points are reached, instead additional CSS classes must be added to change the behavior of the component per breakpoint. The Grid defaults to float left.

AEM uses a concept of Layout Containers to help define the layout of a page. You can think of them as empty components whose only job is to control the layout. Components are then added inside a Layout Container. You can easily identify Layout Containers from other components when you see the .responsivegrid class. A Layout Container known as the Root layout container is typically the outtermost div directly after the body tag. There is nothing special about the Root layout container but it is a common convention to use it as a page-wrapper. It is easily identified with a .root class.

Grid CSS classes can also be applied to the outter div of a component. By default components should take up 100% of the width in whichever container they are dropped in. Use additional Grid classes to control the width of a component within a container. Again because the Grid system is percentage based if a component is set to a width of 6 (50%) and inserted into a layout container that also has a width of 6 (50% of page width), the component would take up only 25% fo the total page width.

In additon to controlling width the Responisve Grid also supports:

  • hiding components based on breakpoint
  • Floating components based on breakpoint
  • offset components to position them on the right.

Default Grid

The .aem-GridColumn--default-xxx where xxx is a number from 1-12. Default simply means there is no breakpoint/media query for these CSS rules. This is easy to conflate with "Desktop" but they are not quite the same, really just anything larger than the Tablet breakpoint.

Tablet and Phone Grid

Mobile and Tablet css classes are then added to container to change the width for respective breakpoints. 

Tablet: .aem-GridColumn--tablet-xxx , Phone: .aem-GridColumn--phone-xxx

Resize your browser to < 1200px to engage the Tablet view and the components should align 2x2. Resizing the browser < 737px will engage the Phone view and components will stack vertically 1x1. View the grid.less and variables.less to see how the breakpoints are set. They can be changed and then the LESS can be re-compiled to utilize a different set of breakpoints.
